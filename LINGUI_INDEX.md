## The Workflow

### Step 1: Write Translatable Content
```tsx
import { Trans } from "@lingui/react";
<Trans>Book Your Appointment</Trans>
```

### Step 2: Extract
```bash
npm run extract
```
Finds all `<Trans>` macros, creates `src/locales/en/messages.po`

### Step 3: Auto-Translate
```bash
python3 scripts/auto-translate.py
```
English → Spanish via MyMemory API, fills `src/locales/es/messages.po`

### Step 4: Compile
```bash
npx lingui compile
```
Creates optimized JSON catalogs for production

### Step 5: Deploy
```bash
npm run build
git push  # Vercel auto-deploys
```
Spanish translations now live

## Key Concepts

### Coexistence (Not Replacement)

Old system (still works):
- `lib/LanguageContext.tsx` — language state
- `lib/translations.ts` — manual translations
- `useLanguage()` hook in components

New system (gradual adoption):
- Lingui `<Trans>` macro for new content
- MyMemory API auto-translation
- Automatic extraction & compilation

**Both work together**. No breaking changes.

### Provider Nesting (Critical)

```
<LanguageProvider>
  <LinguiClientProvider>  ← Must be INSIDE
    {children}
```

The nesting order matters because LinguiClientProvider uses useLanguage().

### Translation Pipeline

```
Code (.tsx)
    ↓
Extract <Trans> macros
    ↓
src/locales/en/messages.po (English source)
    ↓
Auto-translate via MyMemory API
    ↓
src/locales/es/messages.po (Spanish target)
    ↓
Compile to JSON
    ↓
src/locales/*.json (Production catalogs)
    ↓
Build & Deploy
    ↓
Spanish users see translated content
```

### Translation API: MyMemory

- **Free**: 5000 words/day without authentication
- **Rate**: ~2 requests/second (script respects this)
- **Quality**: Basic but usable for initial translations
- **Fallback**: Can swap for LibreTranslate, DeepL, Google Translate

## Troubleshooting Index

### Build Issues
- "SWC Plugin Error" → See `LINGUI_SETUP_GUIDE.md` section "Build fails"
- "Module not found @lingui/..." → Run `npm install`

### Extraction Issues
- "Finds 0 strings" → Normal if no `<Trans>` macros added yet
- "Extract hangs" → Check path configuration in `lingui.config.ts`

### Translation Issues
- "Auto-translate fails" → Check internet connection
- "5000 words exceeded" → Wait 24 hours or use paid API

### Runtime Issues
- "Language doesn't change" → Ensure `LinguiClientProvider` in layout
- "Strings not translating" → Check catalogs compiled (`.json` files exist)

Full troubleshooting: See `LINGUI_SETUP_GUIDE.md`

### Common Questions

**Q: Will this break the existing site?**
A: No. Old system stays active. See "Coexistence" section above.

**Q: Do I need an API key?**
A: No. MyMemory is free with 5000 words/day limit, no auth needed.

**Q: Can I use a better translation API?**
A: Yes. Edit `scripts/auto-translate.py` to use DeepL, LibreTranslate, or Google Translate.

**Q: What about other languages?**
A: Add to `lingui.config.ts` locales array. System supports any language.

**Q: How do I review translations before deploying?**
A: Edit `src/locales/es/messages.po` manually, then compile.
