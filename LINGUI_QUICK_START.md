## Quick Reference: Workflow for Adding Spanish Content

### When you write new translatable content:

```tsx
// Instead of hardcoding strings, use Trans macro:
import { Trans } from "@lingui/react";

export function MyComponent() {
  return <h1><Trans>Book Your Pet's Grooming Today</Trans></h1>;
}
```

### Then run the translation pipeline:

```bash
# Step 1: Extract all Trans macros into English catalog
npm run extract

# Step 2: Auto-translate English → Spanish (MyMemory API, free)
python3 scripts/auto-translate.py

# Step 3: Compile catalogs to optimized JSON
npx lingui compile

# Step 4: Build and deploy
npm run build
npm run deploy  # or npm start, or git push to vercel
```

**That's it.** Spanish translations appear automatically. No manual translation work.

## Important: Activate LinguiProvider

For Lingui to work, you must wrap your app in `app/layout.tsx`:

```tsx
import { LanguageProvider } from "@/lib/LanguageContext";
import { LinguiClientProvider } from "@/components/LinguiProvider";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        <LanguageProvider>
          <LinguiClientProvider>
            {children}
          </LinguiClientProvider>
        </LanguageProvider>
      </body>
    </html>
  );
}
```

## Common Questions

**Q: Will this break the existing site?**
A: No. Old system stays active. Lingui runs alongside.

**Q: Do I need an API key?**
A: No. MyMemory API is free with 5000 words/day limit, no authentication.

**Q: What if the auto-translate quality is poor?**
A: MyMemory is basic but sufficient for initial translations. For production, configure LibreTranslate (self-hosted), DeepL, or Google Translate instead.

**Q: Can I review translations before they go live?**
A: Yes. Edit `src/locales/es/messages.po` manually, then run `npx lingui compile`.

**Q: What about other languages?**
A: Add to `lingui.config.ts` locales array. Currently configured for en, es. Add fr, de, pt, etc. as needed.

## For Developers

When working with translatable strings:

1. **Use `<Trans>` for JSX content**:
   ```tsx
   <Trans>Welcome to The Dog House</Trans>
   ```

2. **Use `i18n._()` for JavaScript strings**:
   ```tsx
   import { i18n } from "@lingui/core";
   const title = i18n._("Dog Grooming Services");
   ```

3. **Run extraction after making changes**:
   ```bash
   npm run extract
   ```

4. **Check the .po file** to see new strings:
   ```
   cat src/locales/en/messages.po
   ```

5. **Run the translation pipeline** to generate Spanish:
   ```bash
   python3 scripts/auto-translate.py
   npx lingui compile
   ```

## Troubleshooting

See `LINGUI_SETUP_GUIDE.md` for detailed troubleshooting section.

Quick fixes:
- **Extract finds 0 strings**: You haven't marked any text with `<Trans>` yet. It's optional until you use it.
- **Build fails**: Check `next.config.ts` has SWC plugin config. If issues, see "SWC plugin error" in guide.
- **Auto-translate fails**: Check internet connection. MyMemory rate limit is 5000 words/day free.
